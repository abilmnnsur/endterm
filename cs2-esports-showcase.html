Single Responsibility

Dependency Inversion

REST API Endpoints

Complete CRUD operations for all entities

Method Endpoint Description
GET /api/articles Get all articles
GET /api/articles/{id} Get article by ID
POST /api/articles Create new article
PUT /api/articles/{id} Update article
DELETE /api/articles/{id} Delete article
GET /api/players Get all players
POST /api/players Create new player
GET /api/teams Get all teams
POST /api/teams Create new team
DELETE /api/inventory/{id} Delete inventory item

REP - Reuse/Release Equivalence

Each package can be released and versioned independently (controller, service, repository)

CCP - Common Closure Principle

Classes that change together are packaged together (all article-related logic in same package)

CRP - Common Reuse Principle

Classes in a package are reused together (repository layer always used with service layer)

Project Reflection

Technical achievements and learning outcomes

Technical Achievements

This project demonstrates proficiency in enterprise-grade Spring Boot development with a focus on clean architecture and maintainability. The implementation follows industry best practices including SOLID principles, proven design patterns, and RESTful API design.

Design Pattern Implementation

The Factory pattern enables flexible article creation without tight coupling. The Builder pattern simplifies complex object construction. Singleton pattern through Spring's IoC container ensures efficient resource management.

Architectural Benefits

The layered architecture provides clear separation of concerns: Controllers handle HTTP requests, Services manage business logic, and Repositories abstract data access. This makes the system highly testable and maintainable.

Database Design

The normalized schema eliminates redundancy while maintaining referential integrity through proper foreign key relationships. JPA/Hibernate handles the object-relational mapping efficiently.
